CSCE 312 Project:
Cache Simulator

By Gage Broberg / January 31, 2022


This project was the end of the year project I did for Dr. Kebo in CSCE 312. I learned a lot in CSCE 312 at Texas A&M with Dr. Kebo (for anyone who this applies to: you should take him!) The course is on the basic of computer architechture, and you begin by learning about logic gates and HDL (hardware description language). You slowly work your way up to higher and higher level concepts, including interactions between the CPU, RAM, and ROM, and programming in assembly language. The course is meant to help programmers like me better understand what is going on at a lower level when we, say, compile and run a program in C++.


The final project was to simulate the interaction between computer cache and RAM, which I did in Python with one of my good friends Armando Cruz. Given a txt file representing the RAM, the requirement was to simulate what happens when a program either reads or writes to memory. Is the memory requested already located in the cache? Do we need to go up a level to retrieve it from RAM and add it to the cache? In this project, we had to answer questions and come up with solutions that mocked the way real hardware and software designers create and implement solutions, and I learned a lot about low-level programming, and computer architecture working on it.


I wish there was a more visual way to show this project off. I love UI design and really wanted to create a nice UI for this project, but unfortunately, the time frame for the project meant that we would have to stick with the classic terminal UI (yuck).
